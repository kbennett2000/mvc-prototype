# Majestic View Church — Kiowa, Colorado

**Live site:** [mvckiowa.com](https://mvckiowa.com)

Majestic View Church (MVC) is a small congregation in Kiowa, Colorado.
They're the inaugural adopter of this template — in fact, the template
itself was generalized from the work done for MVC's site redesign.

## Why MVC?

MVC was a near-perfect fit for what this template was built to prove:

- **Small congregation.** No paid web staff, no marketing department, no budget for a Squarespace plan that bills annually whether the site gets edited or not.
- **A volunteer-led editor team.** People who lead ministries, not people who lead websites. They need to update service times, post announcements, and add a new staff bio without filing a ticket.
- **A pastor who'd been burned before.** The previous site had become unmaintainable; the only person who knew how to update it had moved on. The replacement had to be something the church could own and run themselves, for years, without depending on the original builder.

If the template can serve MVC, it can serve hundreds of churches in the
same shape.

## What was deployed

MVC's site runs the full template feature set:

- **Mobile-first responsive design** with a typography and palette tuned to be reverent, warm, and readable.
- **TinaCMS-driven content** — sermons, staff, elders, ministries, beliefs, events, announcements, and custom pages all editable in the browser by non-developers.
- **Editor-managed navigation** — the nav bar is a CMS field, not a code change.
- **Daily devotional emails** with double-opt-in subscription, per-subscriber reading plans, timezone-aware delivery, and a one-click unsubscribe.
- **Weekly digest emails** that auto-compose from recent announcements, upcoming events, latest sermon, and an optional pastor's note.
- **Subscriber tag architecture** — one email lets a subscriber manage devotional and digest preferences independently.
- **Online giving** via a provider-agnostic integration (Planning Center, Tithely, Pushpay, Subsplash, Stripe, or a custom URL — MVC picks one and the rest stay dormant).
- **Google sign-in for admin pages** with a CMS-managed allowlist so the church can add or remove admins themselves.
- **Vercel hosting** on the free tier, with TinaCloud handling CMS auth and content commits.

The whole stack is recoverable by a new tech volunteer in an afternoon —
which was the original brief.
